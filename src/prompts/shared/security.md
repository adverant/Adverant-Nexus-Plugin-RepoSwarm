# Security Analysis

## Objective
Perform comprehensive security analysis of the codebase.

## Focus Areas

### 1. Input Validation
- User input sanitization
- File upload validation
- URL validation
- SQL injection prevention

### 2. Authentication
- Password handling
- Session management
- Token security
- MFA implementation

### 3. Authorization
- Access control patterns
- Role-based access
- Permission checking
- Privilege escalation risks

### 4. Data Protection
- Encryption at rest
- Encryption in transit
- PII handling
- Data retention

### 5. Dependency Security
- Known vulnerabilities
- Outdated packages
- License compliance
- Supply chain risks

### 6. OWASP Top 10
- Injection flaws
- Broken authentication
- Sensitive data exposure
- XXE vulnerabilities
- Security misconfigurations
- XSS vulnerabilities
- Insecure deserialization
- Component vulnerabilities
- Logging deficiencies
- SSRF vulnerabilities

## Output Format
Provide security findings with CVSS scores and remediation steps.
