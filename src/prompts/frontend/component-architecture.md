# Component Architecture Analysis

## Objective
Analyze the frontend component structure and patterns.

## Focus Areas

### 1. Component Organization
- Atomic design principles
- Component hierarchy
- Reusability patterns
- Props drilling vs context

### 2. State Management
- Local vs global state
- State library usage (Redux, Zustand, etc.)
- State normalization
- Derived state handling

### 3. Performance
- Memoization usage
- Lazy loading implementation
- Bundle splitting
- Render optimization

### 4. Styling Patterns
- CSS-in-JS vs CSS modules
- Design system adherence
- Responsive patterns
- Theme support

### 5. Testing
- Unit test coverage
- Integration tests
- E2E test presence
- Snapshot testing

## Output Format
Provide component tree visualization and optimization recommendations.
