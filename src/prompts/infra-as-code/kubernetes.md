# Kubernetes Configuration Analysis

## Objective
Analyze Kubernetes manifests and Helm charts for best practices.

## Focus Areas

### 1. Resource Configuration
- Resource limits/requests
- Replica counts
- PodDisruptionBudgets
- Affinity/anti-affinity rules

### 2. Security
- Security contexts
- Network policies
- RBAC configuration
- Pod security standards

### 3. Networking
- Service configuration
- Ingress setup
- Service mesh usage
- DNS configuration

### 4. Storage
- PersistentVolumes
- Storage classes
- Backup strategies
- Data persistence

### 5. Observability
- Health probes
- Metrics endpoints
- Logging setup
- Tracing integration

## Output Format
Provide cluster architecture diagram and security recommendations.
