# Terraform Analysis

## Objective
Analyze Terraform infrastructure code for best practices and security.

## Focus Areas

### 1. Module Structure
- Module organization
- Reusability patterns
- Input/output variables
- Local vs remote modules

### 2. State Management
- Remote state backend
- State locking
- Workspace usage
- State file security

### 3. Security
- Secrets management
- IAM policies
- Network security groups
- Encryption configuration

### 4. Resource Organization
- Resource naming
- Tagging strategy
- Dependency management
- Lifecycle rules

### 5. Best Practices
- DRY principles
- Version pinning
- Documentation
- Validation rules

## Output Format
Provide infrastructure diagram and security assessment.
